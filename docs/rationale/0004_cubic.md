# Cubic -- Math Rationale

## Overview

The cubic demo displays a 3D cube with bitmap-font letters textured on each
face. The cube rotates around random axes with periodic changes, and reflections
appear as scaled duplicates. The math covers HSL-to-RGB color conversion,
quaternion rotations for face placement, arbitrary-axis rotation, and reflection
via non-uniform scaling.

## 1. HSL to RGB Conversion

Pastel colors are generated by sampling random hue, moderate saturation, and
high lightness, then converting to RGB:

```rust
fn hsl_to_rgb(h: f32, s: f32, l: f32) -> [u8; 3] {
    let c = (1.0 - (2.0 * l - 1.0).abs()) * s;
    let h_prime = h / 60.0;
    let x = c * (1.0 - (h_prime % 2.0 - 1.0).abs());
    let (r1, g1, b1) = match h_prime as u32 { ... };
    let m = l - c / 2.0;
    [((r1 + m) * 255.0) as u8, ...]
}
```

### Derivation

The HSL cylinder maps:
- **H** (hue): angular position, 0-360 degrees
- **S** (saturation): radial distance from center, 0-1
- **L** (lightness): vertical position, 0-1

**Chroma** is the difference between the max and min RGB channel values:

```
C = (1 - |2L - 1|) * S
```

At `L = 0.5`, `|2L - 1| = 0`, so `C = S` (maximum chroma for that saturation).
At `L = 0` or `L = 1`, `C = 0` (black or white, regardless of saturation).

**Hue prime** divides the 360-degree hue circle into six 60-degree sectors:

```
H' = H / 60
```

Within each sector, one RGB channel is at maximum (`C`), one transitions
linearly (`X`), and one is zero:

```
X = C * (1 - |H' mod 2 - 1|)
```

The `mod 2` creates a triangular wave that peaks at even sector boundaries. The
`match` on `h_prime as u32` selects which channels get `C`, `X`, and `0`.

**Lightness offset** `m = L - C/2` shifts all channels so the midpoint of the
RGB range lands at the desired lightness:

```
R = (R1 + m) * 255
G = (G1 + m) * 255
B = (B1 + m) * 255
```

### Rust idiom

The `match` on `h_prime as u32` is a truncating cast used as a sector index.
Rust's `as u32` truncates toward zero, which is correct here since `h_prime` is
non-negative (hue is in `[0, 360)`). The wildcard arm `_ => (c, 0.0, x)` handles
sector 5 (hue 300-360).

## 2. Quaternion Face Rotations

Bevy's `Plane3d::default()` creates a quad in the XZ plane with normal facing +Y.
To place six faces of a cube, each plane must be rotated so its normal points in
the desired direction.

```rust
fn face_transforms() -> [(Vec3, Quat); 6] {
    let half = CUBE_SIZE / 2.0;
    [
        // Front (+Z): rotate +Y normal toward +Z
        (Vec3::new(0.0, 0.0, half), Quat::from_rotation_x(FRAC_PI_2)),
        // Back (-Z)
        (Vec3::new(0.0, 0.0, -half), Quat::from_rotation_x(-FRAC_PI_2)),
        // Right (+X): rotate +Y toward +X
        (Vec3::new(half, 0.0, 0.0), Quat::from_rotation_z(-FRAC_PI_2)),
        // Left (-X)
        (Vec3::new(-half, 0.0, 0.0), Quat::from_rotation_z(FRAC_PI_2)),
        // Top (+Y): already correct
        (Vec3::new(0.0, half, 0.0), Quat::IDENTITY),
        // Bottom (-Y): 180-degree flip
        (Vec3::new(0.0, -half, 0.0), Quat::from_rotation_x(PI)),
    ]
}
```

### Why these specific rotations?

Starting normal: `+Y = (0, 1, 0)`.

- **Front (+Z)**: Rotating 90 degrees around X maps `+Y -> +Z`. The rotation
  matrix `Rx(pi/2)` sends `(0,1,0) -> (0,0,1)`.
- **Right (+X)**: Rotating -90 degrees around Z maps `+Y -> +X`. The rotation
  matrix `Rz(-pi/2)` sends `(0,1,0) -> (1,0,0)`.
- **Bottom (-Y)**: Rotating 180 degrees around X maps `+Y -> -Y`.

Each quaternion encodes this single-axis rotation. `Quat::from_rotation_x(angle)`
produces:

```
q = (sin(angle/2), 0, 0, cos(angle/2))
```

which is the unit quaternion representing rotation by `angle` around the X axis.

## 3. Arbitrary-Axis Rotation

The cube continuously rotates around a randomly chosen axis:

```rust
fn rotate_cube(mut query: Query<(&mut Transform, &RotationState)>, time: Res<Time>) {
    let angle = rotation.speed * time.delta_secs();
    transform.rotate(Quat::from_axis_angle(rotation.axis, angle));
}
```

### The formula

`Quat::from_axis_angle(axis, angle)` produces:

```
q = (axis * sin(angle/2), cos(angle/2))
```

This is the Euler-Rodrigues parameterization. Any 3D rotation can be described
as rotation by some angle around some axis -- this is the axis-angle theorem.

`Transform::rotate` composes the new rotation with the existing one via
quaternion multiplication: `q_new = q_delta * q_current`. Quaternion
multiplication is non-commutative -- the order matters. Bevy's `rotate` applies
the new rotation in local space (post-multiplication).

### Random axis generation

```rust
fn random_axis(rng: &mut SmallRng) -> Vec3 {
    Vec3::new(
        rng.random_range(-1.0..1.0),
        rng.random_range(-1.0..1.0),
        rng.random_range(-1.0..1.0),
    )
    .normalize_or(Vec3::Y)
}
```

This samples a point in the `[-1, 1]^3` cube and normalizes it. The distribution
is not perfectly uniform on the sphere (corners of the cube are slightly
overrepresented), but for a visual demo the bias is negligible.
`normalize_or(Vec3::Y)` handles the degenerate case where all three components
are near zero.

## 4. Reflection via Non-Uniform Scaling

Reflections are created by spawning duplicate entities with a negative scale on
one axis:

```rust
// Bottom reflection (Y-mirror)
Transform::from_xyz(0.0, -CUBE_Y_OFFSET, 0.0)
    .with_scale(Vec3::new(1.0, -1.0, 1.0))

// Left reflection (X-mirror)
Transform::from_xyz(-CUBE_X_OFFSET, 0.0, 0.0)
    .with_scale(Vec3::new(-1.0, 1.0, 1.0))
```

### The math

A scale of `-1` on an axis reflects all child geometry across the plane
perpendicular to that axis. For the Y-axis mirror:

```
(x, y, z) -> (x, -y, z)
```

This is equivalent to multiplying by the reflection matrix:

```
| 1  0  0 |
| 0 -1  0 |
| 0  0  1 |
```

The reflection entity copies the main cube's rotation each frame:

```rust
reflection_transform.rotation = cube_transform.rotation;
```

Because the negative scale is baked into the transform's `scale` field and
rotation is applied separately, the reflection correctly mirrors the rotated
cube without needing to invert the quaternion.

### Transparency

Reflection materials use `AlphaMode::Blend` with `REFLECTION_ALPHA = 0.1` to
create a faint ghost effect. The `double_sided: true` flag prevents back-face
culling, which would otherwise hide half the reflected faces (the negative scale
reverses winding order).

## 5. Bitmap Font Rasterization

Each face texture is procedurally drawn from an 8x8 bitmap font:

```rust
fn draw_char(data: &mut [u8], ch: usize, offset_x: u32, offset_y: u32, scale: u32, fg: [u8; 3]) {
    let bitmap = &FONT[ch];
    for row in 0..8u32 {
        let bits = bitmap[row as usize];
        for col in 0..8u32 {
            let is_set = (bits >> (7 - col)) & 1 == 1;
```

Each row of a character is stored as a `u8` bitmask. Bit 7 (MSB) is the leftmost
pixel, bit 0 is the rightmost. The expression `(bits >> (7 - col)) & 1` extracts
bit `col` counting from the left.

The scale factor `(TEXTURE_SIZE / 8) - 4 = 28` maps each font pixel to a 28x28
block, producing 224x224 character art centered in the 256x256 texture with
`offset = (256 - 224) / 2 = 16` pixels of padding.

## Summary of Math

| Concept | Formula | Purpose |
|---|---|---|
| HSL chroma | `(1 - abs(2L-1)) * S` | Color intensity from lightness + saturation |
| HSL hue sector | `H / 60`, `match` on sector | Map hue to RGB channel assignment |
| Quaternion from axis-angle | `(axis*sin(a/2), cos(a/2))` | Arbitrary 3D rotation |
| Face normal rotation | `Rx(pi/2)`, `Rz(-pi/2)`, etc. | Reorient +Y plane to cube face |
| Reflection | `scale(-1)` on one axis | Mirror geometry across a plane |
| Bit extraction | `(byte >> (7-col)) & 1` | Read pixel from bitmap font row |

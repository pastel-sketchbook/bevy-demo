# Menu -- Math Rationale

## Overview

The menu demo is a state-machine-driven UI with minimal math. The primary
mathematical element is a countdown timer. This document covers the timer
arithmetic and the state machine design pattern, since the demo's value lies
in its architecture rather than numerical computation.

## 1. Countdown Timer

```rust
fn update_timer(
    time: Res<Time>,
    mut timer: ResMut<GameTimer>,
    mut next_state: ResMut<NextState<GameState>>,
) {
    timer.0.tick(time.delta());
    if timer.0.just_finished() {
        next_state.set(GameState::GameOver);
    }
}
```

### Display remaining time

```rust
let remaining = timer.0.duration().as_secs_f32() - timer.0.elapsed_secs();
**text = format!("{:.1}", remaining.max(0.0));
```

### The formula

```
remaining = total_duration - elapsed
```

`remaining.max(0.0)` clamps to zero to avoid displaying negative values on the
exact frame the timer finishes (a race between the timer system and the UI
update system).

Bevy's `Timer` internally tracks elapsed time as a `Duration` and compares
against the configured duration. `tick(delta)` adds the frame's delta time.
`just_finished()` returns `true` for exactly one frame when the accumulated
time crosses the threshold.

### `{:.1}` format specifier

The format string `{:.1}` displays one decimal place, producing output like
`"9.3"` or `"0.0"`. This is Rust's `Display` formatting for `f32`, using
the precision specifier syntax `:.N` where `N` is the number of decimal digits.

## 2. State Machine (Not Math, but Architecture)

The demo uses Bevy's `States` system with four states:

```rust
#[derive(States, Debug, Clone, Copy, PartialEq, Eq, Hash, Default)]
enum GameState {
    #[default]
    Menu,
    Playing,
    Paused,
    GameOver,
}
```

### Transition graph

```
Menu --[Play]--> Playing
Playing --[Esc]--> Paused
Playing --[Timer]--> GameOver
Paused --[Esc]--> Playing
GameOver --[Restart]--> Playing
GameOver --[MainMenu]--> Menu
```

### Generic cleanup

```rust
fn cleanup<T: Component>(mut commands: Commands, query: Query<Entity, With<T>>) {
    for entity in query.iter() {
        commands.entity(entity).despawn();
    }
}
```

This is a generic Rust function parameterized over any Component type `T`. Each
state's `OnExit` schedule calls `cleanup::<StateEntity>` to despawn all entities
tagged with `StateEntity`. The turbofish syntax `::<StateEntity>` specializes
the generic at the call site -- a core Rust idiom for monomorphization.

## 3. Button Color Interaction

```rust
*bg = match interaction {
    Interaction::Pressed => BackgroundColor(BUTTON_PRESS_COLOR),
    Interaction::Hovered => BackgroundColor(BUTTON_HOVER_COLOR),
    Interaction::None => BackgroundColor(BUTTON_COLOR),
};
```

No math here, but the pattern demonstrates exhaustive matching on Bevy's
`Interaction` enum. The `Changed<Interaction>` query filter ensures the button
system only runs when the interaction state actually changes, avoiding redundant
work.

## Summary

| Concept | Formula | Purpose |
|---|---|---|
| Countdown | `remaining = duration - elapsed` | Display time left in game |
| Clamping | `remaining.max(0.0)` | Prevent negative display on finish frame |
| State machine | `enum GameState` with transitions | Organize game phases with clean teardown |
| Generic cleanup | `cleanup::<T>` monomorphization | Reusable entity despawn per state |
